## THE FIVE DIMENSIONS (ABSOLUTE REFERENCE)

### DIMENSION 1: SCALE (Practitioner Cosmological Sophistication)
- **Primary Label**: Scale
- **Format**: Scale: [1-4]-[Name]
- **Names**: 1-Micro, 2-Meso, 3-Macro, 4-Meta
- **Principle**: Optional advancement, no superiority hierarchy

### DIMENSION 2: LEVEL (Chain Development Progression)
- **Primary Label**: Level
- **Format**: Level: [1-4]-[Name] [Chain Name]
- **Names**: 1-Initial, 2-Intermediate, 3-Advanced, 4-Mastery
- **Chains**: Technology, Sensing, Coherence, Efficacy, Embodiment, Transcendence
- **Principle**: Can differ per chain, advancement requires demonstrated competence

### DIMENSION 3: DEGREE (Lunar Mastery Progression)
- **Primary Label**: Degree
- **Format**: Degree: [1-5]-[Name]
- **Names**: 1-Recognition, 2-Exploration, 3-Commitment, 4-Integration, 5-Transmission
- **Principle**: Coherence-Transcendence specific, orthogonal from Scale/Level

### DIMENSION 4: STAGE (Content Production Platform Maturity)
- **Primary Label**: Stage
- **Format**: Stage: [1-5]-[Name] or Stage: [Name]
- **Names**: 1-Forum, 2-Podium, 3-Amphitheatre, 4-Atrium, 5-Portico
- **Principle**: Sequentially gated (organizational prerequisite), independent from practitioner sophistication

### DIMENSION 5: PHASE (Organizational Business Timeline)
- **Primary Label**: Phase
- **Format**: Phase: [Name] Phase
- **Names**: Foundation, Validation, Scaling, Institute, Infrastructure
- **Principle**: Chronological business timeline, orthogonal from all other dimensions

## ORTHOGONALITY PRINCIPLE (NON-NEGOTIABLE)

These five dimensions are **COMPLETELY INDEPENDENT**. Any combination is valid unless violating internal constraints:

**Internal Constraints That Exist** (enforce these):
- Level advancement requires previous Level mastery
- Degree advancement requires previous Degree mastery
- Stage sequencing (Forum → Podium → Amphitheatre → Atrium → Portico)
- Phase sequencing (Foundation → Validation → Scaling → Institute → Infrastructure)
- Scale selection is free (no ordering)

**Impossible Constraints** (prevent these):
- "Must achieve Scale: 3-Macro before Level: 2-Intermediate" ✗
- "Stage: 5-Portico requires Degree: 5-Transmission" ✗
- "All Levels at 3-Advanced before Phase: Scaling" ✗

## TERMINOLOGY ELIMINATION & STANDARDIZATION

### Eliminate (Find & Replace)
- ~~"Tier"~~ (ambiguous across all meanings) → Use Scale/Stage/Level contextually
- ~~"Mastery Level"~~ (ambiguous five-level reference) → Use Degree: [1-5] or Level: 4-Mastery
- ~~"Phase of development"~~ (confuses business timeline with practitioner progression) → Use Level or Degree
- ~~"Stage"~~ (when referring to chain progression) → Use Level: [1-4]
- ~~"Practice Level"~~ (only if not already using correctly) → Keep if already standardized

### Standardize (Use Exclusively)
- Scale: [1-4]-[Name] (Primary label with name)
- Level: [1-4]-[Name] [Chain] (Always specify chain)
- Degree: [1-5]-[Name] (Lunar systems only)
- Stage: [1-5]-[Name] (Primary label with theatrical name)
- Phase: [Name] Phase (Explicit business timeline reference)

## ANTI-PATTERNS TO PREVENT

### Pattern 1: Terminology Drift
**Prevention**: Every replacement checked against Master Schema
**Detection**: Gate 5 comprehensive search should catch any misses
**Fix**: If drift detected, trace back and consolidate

### Pattern 2: Dimensional Contamination
**Prevention**: Verify dimension before replacing (don't mechanically replace all "Level" with "Level: [1-4]")
**Detection**: Gates specifically check for dimensional confusion
**Fix**: Revert and correct specific instance

### Pattern 3: Incomplete Updates
**Prevention**: Use find-replace with verification, not batch-replace
**Detection**: Gate searches catch missing updates
**Fix**: Second pass on artifact

### Pattern 4: Orthogonality Violation
**Prevention**: Verify no statement suggests impossible constraint before finalizing
**Detection**: Gate validation checks for impossible combinations documented
**Fix**: Rewrite statement to respect orthogonality

### Pattern 5: Inconsistent Scaling Across Artifacts
**Prevention**: Reference Master Schema before every update
**Detection**: Cross-artifact spot-checks catch inconsistency
**Fix**: Standardize to Master Schema immediately

## QUICK REFERENCE: FIND-REPLACE PATTERNS

### Pattern: Generic "Tier" → Context-Specific
```
Before: "I'm at Tier 3"
After: "I'm at Scale: 3-Macro" OR "Level: 3-Advanced in [Chain]" OR "Stage: 3-Amphitheatre"
Rule: Determine context, replace appropriately
```

### Pattern: "Mastery Levels" (5-level) → Degree
```
Before: "Five mastery levels describe progression"
After: "Five Degrees (1-Recognition through 5-Transmission) describe Lunar mastery"
Rule: Always replace with "Degree: [1-5]-[Name]"
```

### Pattern: "Stage" (chain context) → Level
```
Before: "Stage 2 in Technology Chain"
After: "Level: 2-Intermediate in Technology Chain"
Rule: Chain progression always uses Level: [1-4]
```

### Pattern: "Phase" (development context) → Level or Degree
```
Before: "Advanced phase of practice"
After: "Advanced Level (Level: 3-Advanced) of chain practice" OR "Integration phase (Degree: 4-Integration)"
Rule: Never use "Phase" for practitioner progression (Phase is business timeline only)
```

### Pattern: Ensure Primary Label + Name
```
Before: "Level 2" or "Stage 4"
After: "Level: 2-Intermediate [Chain]" or "Stage: 4-Atrium"
Rule: Always include name, not just number
```
