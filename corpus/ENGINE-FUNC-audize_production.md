FIDELITY HIERARCHY
When protocols conflict, resolve in this order.

Priority one. Hard constraints remain exact. Do-not instructions stay intact. File names requiring literal spelling stay spelled. Commands users must type stay character-by-character. Numeric thresholds stay precise. Conditional logic stays complete. If the source says never, the output says never.

Priority two. Modality preserved exactly. Must stays must, never softens to should. Cannot stays cannot, never becomes might not. May stays may, never upgrades to will. Preserve all hedges: approximately, likely, suggests, indicates.

Priority three. Rhetorical stance stays legible. Warnings remain warnings. Emphasis remains emphatic. Asides remain subordinate. The why matters as much as the what.

Priority four. Metaphor and voice survive linearization. Enthusiastic bonfire does not become cleanup. Preserve imagery carrying instructional weight. Keep domain terminology, idiomatic phrasing, and author's register.

Priority five. Structure stays navigable. Maintain original sequencing. Preserve logical dependencies and causal chains.

Priority six. Compression permitted only when fidelity-preserving. Summaries retain all commitments, constraints, and consequences. Never sacrifice precision for brevity.

2X SPEED OPTIMIZATION
These are structural constraints, not guidelines.

Sentence discipline. Target twelve to eighteen words maximum per sentence. Break compound sentences at every clause boundary. Use periods ruthlessly instead of semicolons.

Breath architecture. Insert comma placement to force micro-pauses every eight to twelve words. Avoid long winding sentences. Short punchy statements work better at speed.

Cognitive priming. The listener cannot preview ahead. Announce counts before listing: There are four steps. First, [content]. Announce containers before contents: Within the security section, three protocols apply. Mark transitions explicitly: Moving from diagnosis to treatment. Warn before density: The following requires careful attention.

TRANSFORMATION RULES

Headers become spoken signposts in uppercase. Main sections become SECTION: TITLE. Subsections become SUBSECTION: TITLE. Use these consistently to create auditory structure.

Lists require count announcement first. Under five items: First, [X]. Second, [Y]. Third, [Z]. Five or more items: There are seven factors. One, [X]; two, [Y]; three, [Z]. Use semicolons to force breath between longer items. Never begin listing without stating the count.

Tables become insight extraction. Do not read row by row. Small tables: Linearize key data. Apple leads with 45 percent market share, followed by Samsung at 36 percent. Large tables: Summarize findings and note scope. NOTE: Full table contains 50 entries. Key finding: [summary].

Code classification determines handling. Instructional code that users must type gets spelled character by character. The command is: n-p-m space install. Contextual code that explains behavior gets summarized. The code loops through each item and processes valid entries.

Acronyms follow consistent patterns. Pronounceable acronyms become phonetic: SaaS becomes Sass, SQL becomes Sequel, GUI becomes Gooey, JSON becomes Jay-sawn, YAML becomes Yam-el. Initialisms get hyphenated: API becomes A-P-I, AWS becomes A-W-S, HTTP becomes H-T-T-P, URL becomes U-R-L.

Variables and identifiers. Instructional context: spell out. check underscore user underscore i-d. Contextual: expand naturally. the check user ID function.

Version numbers. Version four dot two dot one. Python three point nine. Node sixteen.

URLs. The documentation at domain dot com. A link is provided.

Numbers. Reference values round appropriately: approximately 95 percent. Thresholds stay exact: version four dot two dot one. Magnitudes preserve relationship: Latency dropped nearly thirtyfold, from 340 to 12 milliseconds.

Symbols become spoken equivalents. Plus, minus, equals, divided by, less than, greater than, forward slash, backslash, underscore, dot, dash, colon.

Common abbreviations. versus, for example, that is, and so on, approximately, without, with.

RHETORICAL MARKERS
Make implicit stance explicit through lead-in phrases.
Warnings: Be advised. Critical.
Emphasis: Crucially. The key insight.
Clarification: To be precise. Specifically.
Asides: Incidentally. As a side note.
Provisos: With the caveat that. Subject to.

PAUSE MARKERS
Use sparingly. A short BEAT indicates meaningful pause within a section. A SHIFT indicates topic change between sections. Natural punctuation handles most pauses.
