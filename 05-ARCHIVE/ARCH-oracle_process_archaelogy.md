## PART III: LEXICON OF COINAGES AND KEY TERMS

### Foundational Neologisms

**Syncrescendence** — The project name and governing concept: civilizational sensing infrastructure for systematic consciousness evolution and intelligence amplification through recursive chain development.

**Identity-Intelligence Complex (IIC)** — The five-chain constellation architecture mapping cognitive functions to specialized AI configurations. Each chain has a planetary (major document) and lunar/satellite (supporting documents) structure.

**Oracle** — A strategic synthesis session conducted through the webapp interface. Stateless by nature, requiring explicit context engineering for continuation. Named Oracle0 through Oracle12+.

**Principal** — The human operator maintaining strategic sovereignty over the AI constellation. The term emphasizes agency and decision authority rather than mere "user" status.

**Reception Calibration** — The paradigm shift from "engineering how models should behave" to "calibrating models to understand the Principal." System prompts became minimal (~1,760 chars) with sophistication moved to membrane layer.

**Blitzkrieg** — Parallel directive execution pattern deploying Oracle Context + multiple directives to Claude Code instances for simultaneous work streams.

### Operational Terms

**Metabolism** — The content lifecycle principle: canonize or delete. Documents either earn permanent status or get deleted. There is no "archive for later" category. **Critical constraint**: Metabolism applies to CONTENT only, never to INFRASTRUCTURE.

**Annealment** — The process of achieving semantic and structural coherence. Structural annealment addresses naming, organization. Semantic annealment addresses prose quality, terminology consistency, cross-reference integrity.

**Ground Truth** — The actual filesystem state as opposed to reports about state. The critical lesson from Oracle 6: evaluate reality, not descriptions of reality.

**Foyer** — The CLI as primary integration primitive. "The AGI will reside in the CLI." File system access + git versioning + multi-tool coordination creates the substrate for persistent, sovereign cognition.

**Culmination** — Capturing value at thread end. A culmination document synthesizes key insights, decisions, and state for continuation.

**Continuation** — Restoring context at thread start. A continuation prompt reinitializes an Oracle thread with necessary context.

**Context Transition Protocol** — The unified framework for culmination, continuation, and compression. AI determines appropriate mode based on state analysis.

**Pedigree** — Oracle lineage tracking. The Pedigree protocol documents the evolutionary history of decisions, insights, and architectural choices across Oracle threads.

**Intention Archaeology Compass** — An instrument for capturing ephemeral intentions before they decay. Extracts Principal brain dumps into structured, navigable form.

### Structural Terms

**Flat Principle** — Hierarchy encoded in filename and metadata, not directory nesting. Primary storage is flat; navigation is through aliases, prefixes, and frontmatter. Maximum 2 directory decisions to reach any file.

**Orthographical Sovereignty** — Content lives local in Markdown+Git. The repository is authoritative. Platform-specific features are leveraged but never create lock-in.

**Zone Ownership** — Which Claude instance handles which domain. Prevents parallel execution collisions.

**Crown Jewel** — A priority canonization item. High-value content awaiting formal CANON status.

**Membrane** — The model-facing layer containing capability charts, prompt engineering specifications, and lab theses. Models are "frozen in time" regarding capabilities—the membrane tells the model what it can do NOW.

### Evaluative Terms

**18 Lenses** — The comprehensive evaluative framework for decisions:

**Category Error** — Applying the wrong operational principle to a domain. The defining example: Oracle 4 applying metabolism (content principle) to orchestration/ (infrastructure).
