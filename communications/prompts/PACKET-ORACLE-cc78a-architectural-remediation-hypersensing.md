### What Broke Historically

The repo grew organically and accumulated:

- duplicate and near-duplicate directories
- stale paths and ghost references
- orphaned prompts and responses
- mixed artifact classes
- partial migrations and inconsistent filing logic

This created what is effectively a scaffold tower-of-Babel problem:

- execution kept advancing
- but the constitutional filing system, playbooks, and communications law did not keep pace

### What CRUSH / Nucleosynthesis Did

To remediate that, a nuclear defragmentation move was performed:

- non-canonical material was flattened into a large corpus
- filename history ceased to be trusted as semantic truth
- material was reclassified by semantic topic rather than legacy folder logic
- a new **neocorpus** was created as a distilled, traversable compendium layer

The resulting hierarchy is now effectively:

- **canon** = hot, sovereign-ratified layer
- **neocorpus** = warm, distilled doctrine layer
- **corpus** = cold provenance layer

That means the repo now has better doctrine to reason from, but still lacks a clean constitutional law for how prompts, responses, playbooks, logs, handoffs, and outputs should be governed.

### The Architectural Symptom

The live stack is now stronger than the filing constitution.

That is the actual disease you are being asked to diagnose and repair.

`constitution -> rendered configs -> runtime -> events -> reconciliation -> ontology`
