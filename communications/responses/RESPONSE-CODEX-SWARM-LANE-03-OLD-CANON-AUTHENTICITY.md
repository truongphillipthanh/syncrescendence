## Returned Content

The old canon should be treated as an authenticity reserve and doctrinal quarry, not as a topology to be restored.

The central conclusion is:

- promote the old canon where it carries durable law about memory sovereignty, context continuity, research rigor, artifact governance, platform sovereignty, and validated implementation practice
- preserve the old canon as pedigree where it carries founding force, lineage, burden-bearing meaning, or mythopoetic coherence but is too tied to the predecessor shell to bind the successor directly
- demote or extract from any region whose authority depended on numbered cosmological topology, five-account IIC structure, dated platform capability assumptions, or long-range modal/business timelines

The most important distinction is this:

- **doctrine** = principles and methods that still govern how the successor shell should think, verify, remember, hand off, and evolve
- **historical logistics** = the old numbering system, celestial navigation grammar, account topology, platform roster, maturity timelines, and shell-specific structural routing

This response does **not** recommend reviving numbered root topology.

5. Where a document mixes durable doctrine with stale topology, promote extracted doctrine and preserve the original as pedigree.
