**3. Prompt / Response / Log Metabolism Model**  
Concrete pathway (direct from WORK-LOOP.md compact step + CONTINUOUS-IMPROVEMENT.md loops):  
1. Raw prompt/response/log lands in communications/ (lane law).  
2. Assessed in same tranche (retros/assessments/ + Oracle-style synthesis per PLAYBOOK.md).  
3. Weekly/directive-close loop identifies recurring pattern (SELF-CRITIQUE-AND-WEEKLY-EVAL pattern).  
4. Pattern formalized as validated-pattern in validated-patterns/ or playbook primitive.  
5. If deterministic → operator; if universally binding → constitutional law via Commander promotion and rehousing receipt.  
6. Old episodic artifact receives supersession pointer and moves to pedigree or archive if no longer live.  
Synthesis: this is the exact "metabolize" obligation stated in WORK-LOOP.md and "compile the win" rule in AGENTS.md. Extrapolation from broader practice: mirrors how mature open-source projects turn issue threads into docs + code.

**4. Office Memory Boundary Model**  
Office-local surfaces (notes, journals, cache, scratch, inbox/outbox) live exclusively in offices/*/ and remain non-authoritative (direct observation from AGENTS.md federal offices and OFFICE-ARTIFACT-LAW-v1.md).  
Shared sync occurs only via explicit handoff to communications/ or executive lanes.  
Promotion thresholds (enforced by Adjudicator gate): (a) explicit assessment score in retro, (b) demonstrated repeatability across ≥2 cycles, (c) formalization into playbook/validated-pattern, (d) Commander receipt confirming no shadow-constitution risk.  
Boundary enforcement: "One Artifact, One Class" validator (ARTIFACT-LAW-VALIDATOR-SPEC-v1.md) + office-envelope physicalization tranches. Synthesis: prevents the predecessor failure mode of local notes silently becoming canon.

**5. Freshness / Staleness / Decay Law**  
Explicit law (not vibes):  
- Stale research → feedstock/ with deprecation marker or pedigree/ archive manifest (KNOWLEDGE-LANE-LAW-v1.md).  
- Outdated platform claims or superseded configs → explicit supersession receipt + pointer to new artifact; old version stays readable but marked historical.  
- Aging playbooks → weekly review scores repeatability; below threshold triggers compaction pass or deprecation (CONTINUOUS-IMPROVEMENT.md PLAYBOOK-COMPACTION-PASS-*).  
- Old but important pedigree artifacts → remain in pedigree/ forever with ROSETTA-TERM-DISPOSITION-MATRIX-v1.md status.  
Freshness trigger: any new work that references an artifact must emit a freshness receipt; absence fails validation. Decay is lawful and loss-aware (rehousing receipt preserves provenance). Direct observation from CONTINUOUS-IMPROVEMENT.md and ROSETTA-INTENT-BACKLOG-TRIAD-RECONSTITUTION-v1.md.

**6. Ontology-As-Memory Assessment**  
Ontology should project only: typed artifacts, control-plane state, connector manifests, surface registries, and relation graphs (direct observation from EXOCORTEX-CONTROL-PLANE-CC91.md, EXOCORTEX-ONTOLOGY-UNIFICATION-CC90.md, and DISSERTATION-SYNCRESCENDENCE-v1.md).  
It must not store source truth—repo files remain canonical.  
Useful when: enabling discovery, audit trails, or typed validation without human parsing.  
Harmful when: creates false confidence (unverified projections treated as law) or bureaucratic overhead (manual ontology upkeep instead of automated receipts).  
Verdict: keep as pure projection layer; regenerate on control-plane updates. Synthesis from exocortex control-plane semantics.

**8. Recommended Compaction Pipeline**  
Autocompaction pipeline (lossless, reversible, auditable):  
1. Capture raw artifact in correct lane (communications/, offices/, runtime/).  
2. Emit assessment/retrospective (WORK-LOOP.md step 4).  
3. Weekly/directive-close loop extracts pattern or anti-pattern (CONTINUOUS-IMPROVEMENT.md).  
4. Formalize into validated-pattern/playbook/operator with explicit provenance link.  
5. Commander/Adjudicator promotion gate issues rehousing receipt and updates ontology projection.  
6. Original artifact receives supersession pointer; moves to pedigree/ or archive manifest if obsolete.  
Provenance preserved via git history + rehousing-receipts/ (pedigree/ structure). Reversibility: receipt always points back to original. Auditability: every compaction traceable in control-plane status and ROSETTA-TERM-DISPOSITION-MATRIX. Semantic continuity: terms bound to live Rosetta constitutional surface. Aggressive compaction is lawful because every step is receipted (direct observation from EXOCORTEX-CONTROL-PLANE-CC91.md verification tranche model and ROSETTA-STONE-EXEGESIS-v1.md compaction-as-metabolism thesis). No data is destroyed; only authority and legibility are compacted.
