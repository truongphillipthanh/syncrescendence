**Current SN assessment**: 3.0x average compression, 74% word reduction, 8.3/10 AI readability. Good proof of concept, but not a compiler output — inconsistent quality, missing metadata, no round-trip guarantee.

**Syncrescript v2 design principles**: Convention over configuration (Rails), concurrent read safety (Elixir), incoherence = compile error (Rust), same data/different views (Notion). Seven principles: single source of truth, convention over configuration, compile-time coherence enforcement, immutable compiled output, concurrent read safety, lossy compression with declared loss, round-trip fidelity for structure.

**Five views of one truth**:
1. **Scripture** — authored prose (current canon, the source)

**Five-stage compiler**: Parse (script) → Validate (script) → Graph (script) → Compress (agent) → Verify (agent). Stages 1-3 are deterministic. Stages 4-5 are agent-assisted.

**The three-layer stack**: Canon (source, being fixed now) → Scaffold (compiler, cleared field ready for construction) → Exocortex (runtime, least mature — Graphiti, dream cycle, live ledger).
