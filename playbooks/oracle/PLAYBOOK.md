## 0. What This Surface Is For

Oracle is the hypersensing and structural-synthesis surface.

Its proper role is:
- form theses
- surface structural patterns and hidden tensions
- pressure-test architecture
- nominate categories, distinctions, and failure modes that local execution may miss

Oracle is not the source of ratification, not the verifier of current product reality, and not the final authority over shell law.

## 1. Native Grain

Oracle’s native grain is:
- long-form synthesis
- broad pattern extraction
- architectural framing
- recursive thesis refinement
- high-context prompting with named anchors

It performs best when:
- the prompt is rich and grounded
- the output contract is explicit
- content-proof requirements are severe
- the question is synthetic rather than merely factual

## 2. What Oracle Is Bad At

Oracle is risky when:
- treated as a current-fact oracle without verification
- allowed to define policy unilaterally
- prompted statelessly or vaguely
- trusted on polished but weakly grounded “verbatim” evidence
- rewarded for rhetorical closure instead of structural accuracy

## 3. Proper Role in Triangulation

Oracle should answer:
- what is the underlying structure?
- what are the blind spots?
- what is the strongest synthesis of the available context?

Oracle should not answer alone:
- is this product capability currently real?
- is this claim officially documented?
- should this be ratified without reconciliation?

That second job belongs to Perplexity/Augur or primary-source verification.

## 4. Prompting Doctrine

The correct Oracle prompt should contain:
- explicit statement of what Syncrescendence is
- actual current-state summary
- anchor files or links
- content-proof requirement
- bounded output structure
- return path and assessment expectation

The correct style is:
- rich context
- named anchors
- ugly content-proof quotes
- own-thesis-first
- explicit constraint against free-floating genericity

## 5. Output Doctrine

Treat Oracle returns as:
- hypothesis-forming
- architecture-shaping
- candidate insight

Do not treat them as:
- self-ratifying truth
- policy without assessment
- reliable current-product verification

Every Oracle response should be:
- landed into repo lineage
- assessed explicitly
- reconciled against constitutional law and implemented reality

## 6. Common Use Cases

Oracle is the right surface for:
- shell redesign
- constitutional diagnosis
- taxonomy nomination
- architecture adjudication
- broad hypersensing over many layers

It is the wrong surface for:
- narrow current-product verification
- exact API/docs questions better answered from primary sources
- low-context fact lookup

## 7. Net Rule

Use Oracle for structural synthesis and architectural pressure.
Always force grounding.
Always assess before ratifying.
