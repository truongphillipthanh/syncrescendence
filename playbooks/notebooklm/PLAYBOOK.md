## 0. What This Surface Is For

NotebookLM is the source-bounded synthesis surface for corpus digestion and structured understanding over curated document sets.

## 1. Native Grain

NotebookLM performs best when:
- source set is explicit and bounded
- query target is synthesis over provided material, not open-web verification
- response is returned as artifact and bridged into repo lineage

## 2. Constraints

- personal NotebookLM remains browser-first; treat it as a web surface unless official stable API reality changes
- do not treat NotebookLM output as constitutional authority
- do not store notebook-internal opaque state as hidden canon

## 3. Dispatch Pattern

Use packetized relay:
- stage packet with `stage_notebooklm_packet.py`
- execute bounded browser relay
- bridge response with `notebooklm_response_bridge.py`
- assess and reconcile before promotion

## 4. Guardrails

- keep NotebookLM distinct from Google model API surfaces
- keep NotebookLM distinct from Perplexity verification role
- preserve pointer/summary discipline instead of bulk raw transcript dumping
