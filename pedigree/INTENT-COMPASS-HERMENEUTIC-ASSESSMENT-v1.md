## 0. Executive Reading

The original Intent Compass was one of the project’s strongest executive instruments and one of its most neglected operational surfaces.

It got three deep things right:

1. intentions are compounding vectors, not isolated todos
2. intention capture and intention steering are different jobs
3. executive truth needs both a cache and a rolling snapshot

It then became too large, too stale, and too weakly drained from its raw queue.

## 1. What Was Good

### 1.1 Intentions as first-class governance objects

The original compass treated intention as a real artifact class rather than as ambient conversation.
That is excellent and should survive.

### 1.2 Category and horizon

The urgent / sprint / backlog / pattern / capture distinction was useful.
It recognized that not all intentions belong on the same time horizon.

### 1.3 Rolling snapshot + cache

The old compass understood that executive truth must be both:

- quick to read
- historically traceable

This remains correct.

### 1.4 Upward binding

The best entries connected intentions to:

- canon
- protocol docs
- implementation work
- projects

That is exactly the kind of cross-lane binding the successor shell needs.

## 2. What Went Wrong

### 2.1 It became too encyclopedic

The compass accreted historical, active, pre-Oracle, pattern, capture, and backfilled intentions into one surface.
That made it rich, but hard to keep alive.

### 2.2 The queue drained too slowly

The summit audit showed a functioning raw capture feed and a broken triage cadence.
That means the problem was not capture.
It was executive maintenance law.

### 2.3 It blurred cache and archive

The same file tried to be:

- active executive steering
- historical archive
- queue summary
- strategic anthropology

Those need to be separated.

## 3. Successor-Shell Consequence

The live successor-shell intent surface should be:

- compact
- current
- intentionally incomplete as archive
- explicitly bound to program and vocabulary

Historical intention archaeology should remain pedigree, not live executive clutter.

## 4. Net Rule

Preserve:

- intentions as first-class artifacts
- category/horizon thinking
- executive cache plus rolling snapshot logic

Do not preserve:

- one mega-file trying to hold all executive history forever
- a capture queue with no enforcement for triage
