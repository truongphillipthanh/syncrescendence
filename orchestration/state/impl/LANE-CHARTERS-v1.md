# Lane Charters v1

These charters physicalize the federal shell inside the canonical `syncrescendence` root.

## Constitutional View

- `communications/`: communications law and lineage
- `executive/`: live executive steering
- `program/`: implementation program and sequencing
- `offices/`: agent- and harness-local offices under federal law
- `playbooks/`: harness- and surface-specific state doctrine
- `operators/`: executable operators and enforcement
- `runtime/`: observational live-state evidence
- `pedigree/`: Rosetta/Intent/Backlog ancestry
- `validated-patterns/`: preserved working exemplars

## Function

The charters exist so directory names are not mere storage bins.

They define:
- what kinds of artifacts belong in each lane
- what role the lane plays in the federal shell
- what should be compacted or promoted elsewhere
- what should never silently become authority

## Operating Consequence

Any future validator or migration tranche should use these charters as lane law.
