# Five-Chain Feed Routing And Handoff Contract v1

**Status**: staged
**Class**: implementation law
**Purpose**: define the lawful routing layer for five-chain intake and downstream handoff without creating parallel raw-intake authorities

## Why This Contract Exists

Campaign 12 widened the inbound constellation to five chain-owned account surfaces.
That widened what can be seen.
It did not lawfully create five equal intake machines.

The current truthful cycle is still Acumen-led:

`captured manifest -> import -> Acumen registry admission -> poll -> triage -> Dawn Brief / verification dossier / repo-side assessment / primary-source queue`

This contract hardens the next step:

- later chains consume derivative artifacts by default
- Acumen remains the governed intake and triage authority
- later chains receive explicit default inbound artifact classes
- each chain has a named stopping point so authority does not silently sprawl

## Routing Law

The five-chain routing layer has three planes.

### 1. Chain-Owned Curation Plane

Each chain may own a curated subscription or source manifest for its teleology.
That is curation authority, not intake adjudication authority.

- Acumen curates reconnaissance and monitoring feeds
- Coherence curates framework, synthesis, and essay-bearing feeds
- Efficacy curates operator, build, and implementation-bearing feeds
- Mastery curates tutorial, pedagogy, and curriculum-bearing feeds
- Transcendence curates meta-pattern, governance, and long-horizon feeds

Captured manifests may widen what Acumen can lawfully consider.
They do not become chain-local raw processing streams by default.

### 2. Governed Intake Plane

Acumen is the sole default intake and triage authority for imported feed material.

Default intake rule:

1. raw candidates route into Acumen-led registry and polling state
2. Acumen emits triage decisions and downstream derivative artifacts
3. later chains consume those derivative artifacts by default

Exception rule:

A later chain may receive a bounded primary-source queue item, dossier, or equivalent source-near artifact only when Acumen or an Acumen-linked repo-side assessment has declared that derivative compression is inadequate for the intended downstream task.

### 3. Downstream Specialization Plane

Coherence, Efficacy, Mastery, and Transcendence are downstream specialization chains.
They are derivative consumers first.
They are not raw-intake peers of Acumen.

## Default Inbound Artifact Classes

### Acumen

Default inbound classes:

- captured feed manifests after import
- governed registry candidates
- polled raw candidate rows
- verification bridge state that remains downstream of triage

Default role:

- intake
- qualification
- triage
- escalation selection

Default stopping points:

- triage decision
- `Dawn Brief`
- verification dossier / Augur packet eligibility
- repo-side assessment
- primary-source queue admission or hold state

### Coherence

Default inbound classes:

- `VERIFIED-SIGNAL-BRIEF-*`
- repo-side assessment artifacts
- `PRIMARY-SOURCE-QUEUE-READOUT-*`
- bounded primary-source queue items with explicit reason codes
- convergent promoted clusters derived from Acumen assessment surfaces

Default role:

- synthesis
- framework extraction
- cross-source integration
- model or thesis formation

Default stopping points:

- synthesis memo
- framework dossier
- comparison lattice
- model update artifact

### Efficacy

Default inbound classes:

- Coherence synthesis memos and framework dossiers
- `VERIFIED-SIGNAL-BRIEF-*` items with direct operational relevance
- repo-side assessments that imply tactical or build action
- bounded primary-source tasks explicitly routed for execution planning

Default role:

- operationalization
- protocol choice
- execution shaping
- backlog or project routing

Default stopping points:

- task packet
- protocol update
- execution plan
- backlog item
- demonstration-domain plan

### Mastery

Default inbound classes:

- Coherence synthesis memos and framework dossiers
- Efficacy-tested procedures and operating notes
- verified exemplars from Acumen downstream products
- bounded teaching queues derived from assessed or synthesized material

Default role:

- explanation
- curriculum design
- pedagogy shaping
- practice-loop formation

Default stopping points:

- lesson module
- curriculum unit
- drill set
- teaching note
- explanatory sequence

### Transcendence

Default inbound classes:

- cross-chain telemetry and yield surfaces
- Coherence synthesis dossiers
- Efficacy outcomes and execution retros
- Mastery curricula and teaching outcomes
- rare high-value primary-source escalations explicitly marked for meta-review

Default role:

- meta-integration
- strategic review
- phase reading
- governance correction

Default stopping points:

- meta-pattern report
- strategic review
- doctrine-pressure signal
- phase-transition readout

## Handoff Law

The default lawful handoff path is:

`chain-owned manifest -> capture/import -> Acumen intake and triage -> assessed derivative artifacts -> downstream specialization`

The highest-value default boundaries are:

1. `captured manifest -> Acumen registry admission`
2. `Acumen triage decision -> verification / assessment / queue state`
3. `repo-side assessment / queue readout -> Coherence`
4. `Coherence synthesis -> Efficacy and Mastery`
5. `cross-chain outputs + telemetry -> Transcendence`

Each boundary changes artifact class before authority changes chain.

## Stopping-Point Law

Acumen should stop once an item has been lawfully placed into awareness, verification, assessed interpretation, or primary-source escalation state.

Coherence should stop at synthesis and framework-bearing artifacts.
It must not reopen raw triage or silently mint execution commitments.

Efficacy should stop at execution-shaping artifacts.
It must not become a hidden raw-source adjudicator except for bounded tasks already escalated through Acumen.

Mastery should stop at transmission-ready artifacts.
It must not metabolize ambient raw feed flow as a substitute for verified exemplars and synthesized understanding.

Transcendence should stop at meta-governance artifacts.
It must remain the downstream integrator, not another intake surface.

## Placement Law

This routing layer is doctrinal.
It does not create later-chain raw-intake runtime trees.

Forbidden default creation surfaces include:

- `runtime/coherence/intake/`
- `runtime/efficacy/intake/`
- `runtime/mastery/intake/`
- `runtime/transcendence/intake/`

If later chains need runtime surfaces, those surfaces must attach to derivative artifact handling, not raw intake replication.

## Anti-Patterns

Forbidden:

1. routing imported raw candidates directly into Coherence, Efficacy, Mastery, or Transcendence as peer intake lanes
2. treating chain-owned manifests as chain-local authority to self-admit raw feed items into the active cycle
3. consuming raw intake streams in later chains when derivative products or bounded queue items already exist
4. allowing later chains to continue past their stopping points and silently absorb another chain's role
5. creating runtime raw-intake directories for later chains before a separate contract explicitly authorizes them
