# Intent and Program Governance v1

**Date**: 2026-03-02  
**Status**: constitutional redesign package  
**Purpose**: define the lawful relationship between executive intention, program backlog, office queues, and transient work queues

## 0. Why This Exists

The old shell proved two things at once:

1. multiple queue and horizon types are real
2. multiple competing authorities are fatal

This document keeps the first insight and forbids the second.

## 2. Queue Types

### Executive intention

Purpose:
- express what matters now
- set horizon and campaign
- bind work upward to values and vocabulary

### Program backlog

Purpose:
- translate executive intent into tranches and active sequencing
- define the practical work queue

### Office queues

Purpose:
- hold local pending work
- stage material before promotion

Office queues are not federal backlog authority.

### Transient queues

Purpose:
- support runtime or relay execution

Transient queues are not executive or program truth.

## 3. Promotion Rules

Material should be promoted:

1. from office queue -> executive when it changes steering
2. from office queue -> program when it becomes sequenced work
3. from transient queue -> communications/runtime when durable evidence is needed

## 4. Retirement Rules

When a queue surface is superseded:

1. mark it pedigree or archive
2. record what authority replaced it
3. do not leave it silently coequal with the new surface

## 5. Anti-Patterns

Forbidden:

1. shadow backlogs
2. orphan raw capture queues with no triage cadence
3. office-local work becoming the real program without promotion
4. historical backlog systems silently treated as live
