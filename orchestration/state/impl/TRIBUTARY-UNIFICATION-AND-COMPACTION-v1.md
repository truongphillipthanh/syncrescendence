**Date**: 2026-03-06  
**Status**: constitutional migration design  
**Purpose**: define the lawful method for threading together the predecessor shell, the pre-schematic semantic archive, and the current successor shell without recreating the topology that forced the reset

---

## 0. Why This Exists

The current shell has built the grid:

- constitutional law
- executive and program lanes
- office topology
- communications law
- operators
- exocortex governance
- ontology projection

But it did so before fully beholding the doctrinal and semantic mass that must eventually inhabit it.

Three tributaries now need to be unified:

1. [syncrescendence](/Users/system/syncrescendence)  
   the live successor shell and current institutional receiving structure
2. [syncrescendence.old](/Users/system/Desktop/syncrescendence.old)  
   the last stable, inhabited shell with strong burden-bearing authenticity
3. [syncrescendence_pre_schematic_design](/Users/system/Desktop/syncrescendence_pre_schematic_design)  
   the highest-density semantic and epistemic archive, including neocanon, neocorpus, and redesign thought

The correct response is not bulk restoration and not continued abstraction.

The correct response is:

**lawful disposition, compaction, and promotion of each tributary into the successor shell according to present authority, future utility, and preserved provenance.**

## 2. Governing Rules

### 2.1 No Wholesale Folder Migration

No root lane from an older tributary should be restored wholesale into the live shell.

Migration happens by:

- artifact class
- semantic family
- authority level
- compaction yield
- current shell fit

### 2.2 The Successor Shell Is the Receiver

The current shell remains the only live receiving institution.

Older tributaries feed:

- pedigree
- doctrine
- references
- patterns
- operators

They do not redefine live root topology.

### 2.3 Provenance Is Never Traded Away for Cleanliness

Culling is allowed.
Untraceable disappearance is not.

Every destructive or shedding act must preserve:

- source path
- tributary
- disposition
- rationale
- extracted lesson if one exists

### 2.4 The Repo Must Stop Acting Like the Primary Residence of All Source Matter

The repo is increasingly:

- law
- control plane
- semantic registry
- compaction engine
- provenance anchor

not a permanent warehouse for every raw source or historical log.

## 3. Lawful Dispositions

Every candidate artifact must receive exactly one primary disposition.

### 3.1 `promote_live_law`

Use when the artifact defines durable shared law that should govern the current shell.

Typical destination:

- [orchestration/state/impl](/Users/system/syncrescendence/orchestration/state/impl)
- root constitutional surfaces

### 3.2 `promote_reference`

Use when the artifact has high semantic value but is not itself live law.

Typical destination:

- [knowledge/canon](/Users/system/syncrescendence/knowledge/canon)
- [knowledge/references](/Users/system/syncrescendence/knowledge/references)

### 3.3 `promote_playbook`

Use when the artifact encodes harness-native or surface-native operational doctrine.

Typical destination:

- [playbooks](/Users/system/syncrescendence/playbooks)

### 3.4 `promote_operator`

Use when the artifact can be turned into deterministic execution.

Typical destination:

- [operators](/Users/system/syncrescendence/operators)

### 3.5 `promote_validated_pattern`

Use when the artifact captures repeatable success but should remain pattern doctrine rather than full law or automation.

Typical destination:

- [validated-patterns](/Users/system/syncrescendence/validated-patterns)

### 3.6 `retain_pedigree`

Use when the artifact is important ancestry, rationale, or authenticity reserve but should not bind the present directly.

Typical destination:

- [pedigree](/Users/system/syncrescendence/pedigree)

### 3.7 `retain_archive_manifest_only`

Use when the artifact itself need not remain resident in the live repo, but its existence and disposition must remain queryable.

Typical destination:

- archive or pedigree manifests
- disposition registry

### 3.8 `externalize_to_exocortex`

Use when the artifact belongs on a connected surface or external substrate more naturally than in the live repo.

The repo retains:

- pointer
- receipt
- synopsis
- promotion history

### 3.9 `cull_with_receipt`

Use only when:

- epistemic yield is low
- operational yield is low
- provenance sensitivity is low
- or the lesson has already been cleanly extracted elsewhere

No cull occurs without a recorded receipt.

## 6. Source and Log Shedding Law

### 6.1 Sources

Old news, stale platform chatter, and historically temporary source captures should not be retained by default in the live repo.

Retention test:

- does it still change the shell’s epistemology?
- does it still change practice?
- does it still serve as irreplaceable provenance?

If not, it should be externalized or culled with receipt.

### 6.2 Logs

Logs are not sacred because they happened.
They are sacred only insofar as they preserve:

- evidence
- rationale
- failure pattern
- reusable lesson

If no lesson can be mined and provenance sensitivity is low, the log should be culled after receipt and extraction.

### 6.3 Lessons

When a lesson is extracted, it should promote to one of:

- law
- playbook
- validated pattern
- operator
- executive delta
- program delta

This is the compaction rule that prevents heroics from remaining isolated anecdotes.

## 7. Exocortex and Ontology

The current shell changes the migration problem because the exocortex is now a real governed layer.

That means the repo can increasingly hold:

- semantic authority
- execution doctrine
- receipts
- manifests
- projections

while connected systems hold more of the raw operational or content surface.

### 7.1 Exocortex

The exocortex should absorb increasing raw-surface burden when:

- the native integration is stronger than local file residency
- the content is operationally live but not canonically central
- provenance can still be preserved through receipts and manifests

### 7.2 Ontology

Ontology is the unifier of cognitive organs only if it remains projection and registry rather than usurping law.

Its role is to:

- expose relations
- type entities
- surface state
- make the wider shell queryable

It must not become a second constitution or a silent replacement for Rosetta.
