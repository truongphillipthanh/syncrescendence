## Design Rule

Root files should exist only when they improve one of these:

1. session orientation
2. coordination discipline
3. continuity
4. compaction pressure
5. root-level harness ergonomics

## What Does Not Return to Root

The successor shell deliberately does not restore certain predecessor root files:

- `ACTIVE-TASKS.md`
  - rejected because it would shadow the live program lane
- platform extension files as independent root authorities
  - rejected because harness-specific detail should compact into playbooks and generated surfaces
- dashed logistics
  - rejected because they were a topological source of drift and shell friction
