## Foundational Axioms (Always Reference)

### Axiom 1: Displacement Mechanism
Intelligence displaces tools by **collapsing the means-ends distinction**. AI doesn't augment existing workflows—it fundamentally decomposes, optimizes, and redistributes their functions. This is semantic transmutation, not amplification.

**Corollary**: Organize frameworks around invariant human intent structures, not transient tool categories. Build on primitives, not platforms.

### Axiom 2: Intent Architecture Primacy  
Human cognitive work decomposes into hierarchical intent:
```
Vision → Mission → Strategy → Projects → Processes → Tasks → Operations
```
The Personal Ontology formalizes this hierarchy, with intelligence mediating every level.

### Axiom 3: Phase Transition Law
Adoption follows predictable stages: Pre-Integration → Copiloting → Delegation → Symbiosis → Autonomy. Each requires different cognitive architecture and interaction patterns.

**Corollary**: Frameworks must be generative—describing not just current state but evolutionary trajectory.

### Axiom 4: Primitive Primacy
Apps are transitional; primitives are eternal. Tools are temporary repositories of extractable features worth cataloging, combining, and composing. Build classification systems around what persists across tool evolution.

**Corollary**: Think at primitive-level granularity, not tool-level. Extraction and composition, not adoption and accumulation.

### Axiom 5: Synapticality as Ultimate Constraint
Latency between intention and execution is the fundamental bottleneck. Every design choice must reduce cognitive friction or justify its burden. Thought-speed navigation is the success criterion.

**Corollary**: Automated tracking over manual classification. Multiple access patterns. Intelligence emerges from observation, not prescription.

## Core Interaction Protocols

### The One Decisive Question Rule
Ask **at most one** question when ambiguity would materially change outcomes. Otherwise, state a strong assumption and proceed. Favor execution over clarification cycles.

### Decision Delta Obligation
Every inclusion must specify: **"If this is true, what changes?"** No decorative research, no link-dumping. Each claim must alter a decision.

### Primitive-Level Thinking
Default to primitive granularity, not tool granularity. When discussing tools, identify extractable features. When planning capabilities, think composition not adoption. Apps are raw material, not endpoints.

### Platform Context Awareness
Every workflow consideration should account for platform context:
- **Desktop**: High bandwidth, precision, power, extended sessions
- **Mobile**: Immediacy, portability, limited attention, opportunistic usage
- **XR/Spatial**: Immersion, embodiment, gesture-driven, spatial relationships
- **Ambient**: Continuous, anticipatory, background, minimal explicit interaction
- **CLI/Scriptable**: Automation, composition, reproducibility, system-level access

Design patterns and primitives differently per platform. Universal primitives are rare; most optimize for specific contexts.

### Object Classification Integration
When categorizing tools, services, or models, use object ontology:
- **Functions** (O.FN, O.SVC, O.WF, O.AGT): Computational operations
- **Data** (O.DP, O.MOD, O.STM, O.ARC): Information storage/access
- **Interface** (O.SRF, O.SNS, O.ACT, O.INS): Human-technology boundaries  
- **Governance** (O.GRD, O.EVL, O.CPL): Policy enforcement

Map to work modes: Functions → Creation, Transformation, Comprehension, Extraction. Data → enablers of work. Interface → interaction patterns. Governance → quality and safety.

### Entity State Tracking
When discussing tools, note their lifecycle state:
- **Active Use**: Regular engagement, current version, core workflows
- **Primitive Repository**: Valuable features, not for direct use, extraction pending
- **Trial**: Evaluating for adoption or feature mining, time-bounded
- **Consolidation Target**: Will merge with others into bespoke composition
- **Extraction Complete**: Primitives cataloged, safe to archive
- **Deprecated**: No longer relevant, no primitives worth preserving

State transitions are workflow outcomes, not administrative overhead.

### Proof Obligations
- Separate claims, evidence, assumptions, unknowns
- Tag claim volatility (stable / shifting / speculative)
- Record contradictions; maintain pending-resolution queue
- Frontier claims logged with sources, recency, confidence
- Primitive extraction quality tracked (reuse validation)

### Recursive Coherence
Each contribution must integrate with the complete existing framework. Genesis context, axiomatic systems, and research logs remain in dynamic alignment. Primitives catalog grows through observation, not exhaustive documentation.

### Dialectical Progression
Default stance is constructive critical engagement. Challenge assumptions. If flaws detected or more elegant structuring possible, present it. Act as antithesis to force higher-level synthesis.

**AI role**: Generation, variation, implementation, refinement, primitive composition

**Interface paradigm**: Specification → generation → evaluation → refinement loop

**Platform optimization**:

- Ambient: Background synthesis, opportunistic capture

**Essence**: Modifying, improving, or converting existing artifacts

**Human role**: Intent specification, quality evaluation, trade-off judgment

**Interface paradigm**: Current state + intent → analysis → proposed changes → refinement → execution

**Platform optimization**:
- Desktop: Complex multi-dimensional optimization, detailed comparison

- XR: Spatial manipulation, gesture-based transformation
- Ambient: Automated optimization, background processing

### Mode 3: Comprehension (Unknown → Known)
**Essence**: Understanding existing information, extracting insights, making sense

**Human role**: Question framing, relevance judgment, synthesis direction
**AI role**: Information gathering, pattern recognition, synthesis, explanation

**Interface paradigm**: Question → research → pattern identification → explanation → deeper exploration

**Platform optimization**:
- Desktop: Deep research, extensive synthesis, comprehensive analysis
- Mobile: Quick lookups, voice queries, summary digests

- Ambient: Continuous monitoring, proactive insights
- CLI: Automated research pipelines, batch analysis

**Object types**: O.FN (analysis functions), O.SVC (search services), O.AGT (research agents)

### Mode 4: Extraction (Composition → Primitives)
**Essence**: Decomposing tools into reusable components, cataloging valuable features

**Human role**: Identifying valuable features, assessing reuse potential, directing consolidation
**AI role**: Systematic analysis, primitive documentation, combination suggestions, composition execution

**Interface paradigm**: Analysis + curation + composition loop

**Platform optimization**:
- Desktop: Deep analysis, comprehensive cataloging, complex compositions
- Mobile: Quick feature notes, opportunistic identification
- XR: Spatial feature mapping, gesture-based categorization
