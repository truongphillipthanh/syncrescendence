## IV. ACCOUNTING FOR RELIABILITY TRADEOFFS

Instead of avoiding features due to spottiness, we account for the spottiness:

### Claude Thread Limits
**Tradeoff**: Hard interruption mid-thought, no compaction
**Mitigation**: 
- Offload static context to Project Files (preserve interaction volume)
- Use past chat search to recover prior reasoning
- Produce self-contained artifacts before thread exhaustion
- Handoff token generated at natural breakpoints, not emergency

### ChatGPT Context Rot
**Tradeoff**: Older context decays, antithetical to holistic framing
**Mitigation**:
- Use ChatGPT for bounded ideation windows, not long-term coherence
- Canvas mode preserves document state across context rot
- Project-Only Memory prevents global leakage
- Accept: ChatGPT excels at bursts, not continuity

### Gemini Isolation
**Tradeoff**: No project-scoped isolation
**Mitigation**:
- Use Drive sync for structured input/output
- Treat Gemini as analysis engine, not conversation partner
- NotebookLM for one-shot digestion tasks

### Grok Safety/Feature Gaps
**Tradeoff**: Regulatory uncertainty, limited beyond X
**Mitigation**:
- Use Grok for what it excels at: EQ, speed, social sensing
- Don't rely on Grok for critical path work
- Accept: Grok is complementary, not foundational

### Perplexity Statelessness
**Tradeoff**: No memory, no agency
**Mitigation**:
- Use Perplexity for point-in-time verification
- Don't expect continuity—each query is fresh
- Strength IS the weakness: no memory means no drift
