## Pattern

- do not run context to exhaustion
- compact proactively at natural breakpoints
- keep critical state in files, not only in model context
- prefer explicit focus instructions when compacting
- treat degraded recall, looping, and contradiction as compaction signals, not something to push through

## Shell Consequence

This pattern validates:

- root cockpit files
- office-local memory surfaces
- explicit handoffs
- externalized plans and receipts

It also strengthens the successor-shell view that compaction is institutional metabolism, not just context-window triage.
