# Implementation Tranche T — Tributary Unification and Compaction

**Tranche**: T  
**Intent bindings**: `INT-SHELL-001`, `INT-SHELL-003`, `INT-SHELL-005`

## Purpose

Thread together the three major tributaries of Syncrescendence:

- the live successor shell
- the older inhabited shell
- the pre-schematic semantic archive

without recreating predecessor topology or allowing semantic volume to flood the live shell ungoverned.

## Tasks

1. ratify tributary authority split:
   - current shell = live receiving structure
   - `syncrescendence.old` = authenticity and burden memory
   - `syncrescendence_pre_schematic_design` = highest-density epistemic archive
2. build a disposition registry for migration candidates with one primary disposition per artifact:
   - `promote_live_law`
   - `promote_reference`
   - `promote_playbook`
   - `promote_operator`
   - `promote_validated_pattern`
   - `retain_pedigree`
   - `retain_archive_manifest_only`
   - `externalize_to_exocortex`
   - `cull_with_receipt`
3. execute migration in high-yield order:
   - semantic constitution
   - operational wisdom
   - executive meaning
   - office habitation
   - source shedding
4. define and apply folder-to-lane mappings for:
   - `01-CANON`
   - `05-SIGMA`
   - `00-ORCHESTRATION`
   - `-SOVEREIGN`
   - `agents`
   - `04-SOURCES`
5. define source/log shedding law so obsolete material can be compacted or culled without destroying provenance
6. bind exocortex and ontology into the migration program so the repo trends toward control-plane, registry, and compaction engine rather than undifferentiated warehouse

## Promotion / Completion Criteria

- tributary unification law exists as a shared implementation reference
- the live backlog names tributary unification as an active tranche
- disposition classes are explicit and bounded
- folder-to-lane mapping is explicit for the major predecessor roots
- source/log shedding is governed by receipt-preserving law rather than ad hoc cleanup
- office burdens for the unification program are explicit enough for multi-agent execution
