## V. SEMANTIC NOTATION FORMAT

### Block Structure
```markdown
TERM BlockName:
sutra: "Memorable essence in ≤100 chars—crafted, not truncated"
gloss:
    2-4 sentences contextualizing for Ψ architecture.
    Adds value beyond source. References related concepts.
spec:
    type: [TERM|NORM|PROC|REF]
    source: original_file_name
    applies_to: [list, of, domains]
    key_insight: extracted_pattern
    refs: [[Related-File-1]], [[Related-File-2]]
end
```

### Block Types
- **TERM**: Concept definitions (skills, MCP, worktrees)
- **NORM**: Behavioral rules and constraints (75% rule, anti-patterns)
- **PROC**: Procedures and workflows (handoff protocol, Ralph pattern)
- **REF**: Reference material (platform comparisons, capability matrices)

### Quality Checklist
- [ ] Sutra is NEW SYNTHESIS (not first N characters)
- [ ] Sutra captures essence memorably (could be an aphorism)
- [ ] Gloss adds context for Ψ (not verbatim copy)
- [ ] Spec contains only typed fields (no prose sentences)
- [ ] Cross-references use `[[FileName]]` format
- [ ] Block names are PascalCase identifiers
